# Session Prompt — Onboarding

Use this prompt when a new developer or new agent joins **OSS Project Plan**.

---

Welcome to **OSS Project Plan**.

To get oriented, read these files in order. Stop after each one and make
sure you understand it before moving on.

1. `.handover/context/overview.md` — what this project is and why it exists.
2. `.handover/context/architecture.md` — how it is built.
3. `.handover/context/decisions.md` — why it is built that way.
4. `.handover/context/constraints.md` — the hard rules.
5. `.handover/context/risks.md` — what is still uncertain.
6. `.handover/work/spec.md` — what is currently being built.
7. `.handover/work/tasks.md` and `.handover/work/milestones.md` — the
   current work plan.
8. `.handover/standards/` (all four files) — the rules for changes.

When you are ready to start contributing:

- Pick a task from `.handover/work/tasks.md`.
- Use `.handover/prompts/implement.md` to start the work.
- Use `.handover/prompts/review.md` before opening a pull request.
