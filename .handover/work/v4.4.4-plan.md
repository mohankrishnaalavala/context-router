# v4.4.4 — Honesty release

**Status:** in progress
**Owner:** main session
**Branch:** `develop` → PR to `main`
**Goal:** Make context-router credible to skeptical engineers and reproducible by contributors. Single PR.

## Context

Prior session benchmarked v4.4.3 against kubernetes (197K symbols) and got 3/3 rank-1 with 91.5% fewer tokens than `code-review-graph`. Judge review found two weaknesses:

1. The k8s benchmark used **review mode at the fix SHA** — `git diff HEAD HEAD~1` already contains the GT files as `changed_file` candidates at confidence 0.95. Near-tautological: not measuring retrieval, measuring "does the tool not throw away input."
2. The "91% fewer tokens" comparison vs `code-review-graph` is workload-mismatched. Their 1,506-token average wasn't on the same tasks.

Plus a real product gap: implement-mode on >10K-symbol repos with no diff anchor returns noise, because `SymbolRepository.get_all` caps at 10K rows with no `ORDER BY`. Same as the django-t3 follow-up flagged in `BENCHMARKS.md`.

Plus a packaging gap: only `py/java/ts/cs/yaml` analyzers ship in the wheel; `language-go/rust/ruby/php/sql` exist in `packages/` but have no entry-point registration and no force-include — first-time installs on Go/Rust repos silently index nothing.

### Phase 4 — FTS5 implement-mode candidate retrieval (Days 5–6)

The right fix for the >10K symbol cap is FTS5-anchored retrieval, not a token-seeder over the truncated slice.

- New migration in `packages/storage-sqlite/src/storage_sqlite/migrations/`:
  ```sql
  CREATE VIRTUAL TABLE symbols_fts USING fts5(
    name, qualified_name, file_path,
    content='symbols',
    tokenize='porter unicode61'
  );
  ```
- Triggers to keep FTS in sync with symbols table on insert/update/delete
- `SymbolRepository.search_fts(query: str, limit: int = 200) -> list[Symbol]`
- `Orchestrator._implement_candidates`: union FTS top-200 with `get_all` 10K slice, dedupe, preserve v4.4.3 source-prior multiplier
- Tests:
  - Unit test for FTS migration + repository method
  - Integration test: implement-mode at k8s parent-SHA + `"kubelet dra unprepareResources error handling"` returns `pkg/kubelet/cm/dra/manager.go` in top-3
  - Regression: suite A (gin/actix-web/django) and suite B (gson/requests/zod) F1 must not drop
