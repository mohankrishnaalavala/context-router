# Design Docs

Per-release design specifications. Each file is written **before** implementation begins and serves as the agent's authoritative "what are we building and why" reference for that release.

## How to use

- **Starting v4.3 work?** Read `v4.3-staleness-federation.md` first, then open `.handover/prompts/implement.md` for the session sequence.
- **Planning a new release?** Copy the structure from the most recent design doc. Fill in sections 1–6 (Outcome, Why, Scope, DoD, ADRs, Risks) before writing any code.
- **Just landed in this repo?** Start with `CLAUDE.md` at the project root — it has the full orientation map.
