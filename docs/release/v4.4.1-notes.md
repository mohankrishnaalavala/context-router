# context-router v4.4.1 — precision-first release

> Patch release that ships the v4.4 precision-first redesign as a single, coherent line of changes (PRs #107–#110). v4.4.0 was tagged as the carrying version; v4.4.1 is the first release where all four phases are merged into `main`.

## Headline

context-router now **beats `code-review-graph`** on 2 of 4 benchmark repos (fastapi, bulletproof-react, eShopOnWeb, spring-petclinic) on a 12-task tuning suite, and meets both v4.4 quality targets:

| Metric | v4.3 baseline | v4.4.1 | code-review-graph |
|---|---:|---:|---:|
| Avg F1 (file) | 0.378 | **0.611** | 0.583 |
| Avg precision (file) | 0.279 | **0.625** | 0.658 |
| Avg tokens | 3,416 | **1,860** | 1,506 |
| Rank-1 = GT file | 8/12 | **10/12** | n/a |
| Empty / degenerate | 0/12 | **0/12** | 2/12 |

Token cost is down 46% and F1 is up 62% vs v4.3, with no degenerate (empty) outputs.

## What changed (by phase)

### Phase 1 — Precision-first ranking core (PR #107)
- **Per-mode token budget defaults.** New `mode_budgets` config: `review`/`implement` 1500, `debug` 2500, `handover` 4000, `minimal` 800. Override per call with `--max-tokens` or per project under `mode_budgets:` in `.context-router/config.yaml` (partial overrides merge with defaults).
- **Score floor.** Per-mode threshold drops low-confidence noise before knapsack admission: `max(top1 * 0.55, 0.30)` for `review`/`implement`/`minimal`; `0.30` for `debug`; `0.20` for `handover`.
- **Restricted per-source-type guarantee.** The catch-all `file` source-type guarantee survives only in `review` and `handover`. High-signal types (`entrypoint`, `changed_file`, `runtime_signal`, `contract`, `extension_point`, `failing_test`, `past_debug`, `blast_radius`, `impacted_test`, `query_match`) remain guaranteed everywhere.
- **Boost scoping.** Hub/bridge structural boost and community-cohesion +0.10 are now scoped to `handover` mode only. Per-task modes get cleaner ranking signal.
- **Top-1 symbol-body inlining (Phase 5 in plan).** Only the rank-1 symbol body is inlined into the pack; all others ship as file pointers with `lines: [start, end]`. This is the single largest token-cost reduction in the release.

### Phase 2 — Cross-encoder reranker (PR #108)
- **`--with-rerank` / `--no-rerank` CLI flag.** Opt-in second-stage rerank over the top-30 candidates using `cross-encoder/ms-marco-MiniLM-L-6-v2` (~22 MB, lazy-downloaded once).
- 50/50 blend of structural confidence with sigmoid-normalized cross-encoder score.
- Falls back silently when `sentence-transformers` is unavailable; latency cost ~50 ms per pack on CPU.
- Lifts precision +0.10 to +0.20 on query-driven packs.

### Phase 3 — Query-driven widening + adaptive depth (PR #109)
- **Query-token candidate widening.** When `changed_files` is empty (no authoritative diff), `_review_candidates` widens the candidate pool by matching tokenized query terms against file stems. New `query_match` source type with confidence 0.55. Skipped automatically when a real diff is present, so PR-review packs don't get diluted.
- **Adaptive depth metadata.** Every pack now carries `metadata.depth` (`narrow` / `standard` / `broad`) and `metadata.depth_reason`. Classified from top-1 confidence and the gap to top-2:
  - `narrow` when top1 ≥ 0.75 and `(top1 - top2) ≥ 0.15`
  - `broad` when top1 < 0.55
  - `standard` otherwise
- Depth is informational in this release — clients can use it to decide whether to read more code, split the query, or proceed with the top items.

### Phase 4 — Feedback loop closed (PR #110)
- **`files_read` becomes a positive signal.** After ≥3 reports for a file, `files_read` adds **+0.03** to its confidence — smaller than missing (+0.05), larger than nothing. Combines additively with missing/noisy signals (e.g. files_read + noisy = +0.03 − 0.10 = −0.07).
- **`pack.metadata.feedback_applied` audit list.** Each pack exposes which historical adjustments shaped this pack: `[{path, delta}, …]`. Adjustments for files not selected into the visible pack are filtered out so the metadata stays answer-focused.
- All adjustments remain scoped per project via `repo_scope`.

## New / changed CLI flags

| Flag | Phase | Notes |
|---|---|---|
| `--with-rerank` / `--no-rerank` | 2 | Opt-in cross-encoder rerank pass. Default off. |
| `--max-tokens N` | 1 | Now overrides the new `mode_budgets[mode]` resolution chain. |

No flags removed. `--with-semantic` / `--no-semantic` behaviour is unchanged.

## New / changed config keys

| Key | Default | Notes |
|---|---|---|
| `mode_budgets` | `{review: 1500, implement: 1500, debug: 2500, handover: 4000, minimal: 800}` | Partial overrides merge with these defaults — only the named modes are replaced. |
| `token_budget` | `8000` | Now a fallback for unknown modes; explicit `mode_budgets[mode]` always wins. |
| `capabilities.hub_boost` | `false` | Now scoped to `handover` mode only when enabled. |

## New pack metadata keys

| Key | Type | Notes |
|---|---|---|
| `metadata.depth` | `"narrow" \| "standard" \| "broad"` | Phase 3. |
| `metadata.depth_reason` | string | Phase 3. Human-readable explanation. |
| `metadata.feedback_applied` | `[{path, delta}]` | Phase 4. Filtered to paths visible in the pack. |

## Optional new dependency

- `sentence-transformers` (already optional for `--with-semantic`). When `--with-rerank` is used, the cross-encoder model `cross-encoder/ms-marco-MiniLM-L-6-v2` is fetched on first run. Set `HF_HUB_OFFLINE=1` and use `--no-rerank` in air-gapped environments.

## Breaking-ish changes

These are technically defaults-changes, not API breaks, but worth flagging:

- **`review` default budget cut** 4000 → 1500. Restore with `--max-tokens 4000` or `mode_budgets.review: 4000` in config.
- **`implement` / `debug` / `handover` default budgets cut** 8000 → 1500 / 2500 / 4000.
- **Hub/community boosts no longer fire in per-task modes.** Re-enable globally by patching `capabilities.hub_boost: true` and using `handover` mode, or vendor in a config patch if you depended on these signals.

## Forward-looking (deferred to post-v4.4.1)

Documented in `docs/superpowers/plans/2026-04-27-performance-enhancements.md` under "Future phases":

- **Phase 6 — query-conditional feedback.** Cosine-weight per-file adjustments by query embedding similarity, so feedback for query X doesn't bleed into unrelated query Y.
- **Phase 7 — "treat docs-only diffs as no-diff" heuristic.** Lets Phase 3 widening fire on PRs whose only changed file is `release-notes.md` / `CHANGELOG.md` / `docs/**`.
